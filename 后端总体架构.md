# AI模拟面试与能力提升平台 - 后端总体架构设计

## 1. 系统概述

本系统是一个面向计算机专业学生的AI模拟面试平台,核心目标是提供针对特定技术岗位的智能面试训练、多维度评估反馈和个性化能力提升建议。

## 2. 技术栈选型

### 2.1 核心技术
- **开发语言**: Python 3.10+
- **Web框架**: FastAPI (异步高性能)
- **向量数据库**: Chroma DB (支持RAG检索)
- **关系数据库**: PostgreSQL (主数据存储)
- **缓存层**: Redis (会话管理、消息队列)
- **消息队列**: Redis Pub/Sub / RabbitMQ (微服务通信)
- **认证**: JWT (JSON Web Token)
- **API文档**: OpenAPI 3.0 (FastAPI自动生成)

### 2.2 AI相关技术
- **大语言模型**: 支持多种LLM接入(OpenAI、Claude、国产大模型等)
- **向量化**: Sentence Transformers / OpenAI Embeddings
- **RAG框架**: LangChain / LlamaIndex
- **数字人框架**: OpenAvatarChat (面试交互微服务基础)

### 3.2 模块划分

#### 3.2.1 主服务模块 (FastAPI)

**1. 认证授权模块 (auth_service)**
- 用户注册、登录、登出
- JWT Token生成与验证
- 权限控制(RBAC)
- 多登录方式支持(预留)

**2. 用户管理模块 (user_service)**
- 用户信息CRUD
- 用户画像管理
- 学习历史记录
- 个人能力档案

**3. 岗位管理模块 (position_service)**
- 岗位信息维护
- 岗位技能要求配置
- 岗位题库关联
- 岗位知识库关联

**4. 题库管理模块 (question_service)**
- 题目CRUD操作
- 题目分类与标签
- 题目难度分级
- 题目与岗位关联
- 题目推荐算法

**5. 面试会话管理模块 (interview_service)**
- 面试会话创建与管理
- 面试状态跟踪
- 面试记录存储
- 与AI数字人微服务通信

**6. 评估分析模块 (evaluation_service)**
- 答案内容分析
- 技术能力评估
- 表达能力评估
- 综合评分计算
- 多维度评估报告生成

**7. 知识库模块 (knowledge_service)**
- 知识文档管理
- 文档向量化处理
- RAG检索服务
- 知识点与岗位关联

**8. 报告生成模块 (report_service)**
- 面试报告生成
- 能力雷达图生成
- 历史对比分析
- 报告导出功能

**9. 能力提升模块 (improvement_service)**
- 个性化学习路径推荐
- 薄弱点分析
- 学习资源推荐
- 练习题推荐

#### 3.2.2 AI数字人面试微服务 (interview_avatar_service)

基于OpenAvatarChat框架的独立微服务,负责面试交互的核心逻辑。

**核心组件:**
- **对话引擎**: 管理多轮对话流程
- **自定义Handler集合**:
  - `InterviewQuestionHandler`: 题目推送与管理
  - `AnswerEvaluationHandler`: 答案收集与初步评估触发
  - `InterviewFlowHandler`: 面试流程控制
  - `ContextManagementHandler`: 上下文管理
  - `EmotionAnalysisPlugin`: 情感分析预留接口(未来扩展)
- **数字人渲染**: OpenAvatarChat原生能力

## 4. 服务间通信方式

### 4.1 主服务与AI数字人微服务通信

**方案一: HTTP/HTTPS + WebSocket (推荐)**
- 控制指令: HTTP RESTful API
- 实时对话: WebSocket双向通信
- 适用场景: 面试开始、结束、状态查询等

**方案二: Redis Pub/Sub**
- 异步消息传递
- 事件驱动架构
- 适用场景: 评估结果通知、题目推送等

**通信协议设计:**
```json
// 面试开始请求
{
  "action": "start_interview",
  "session_id": "uuid",
  "user_id": "user_uuid",
  "position_id": "position_uuid",
  "config": {
    "difficulty": "medium",
    "duration": 30,
    "question_count": 10
  }
}

// 题目推送
{
  "action": "push_question",
  "session_id": "uuid",
  "question": {
    "id": "q_uuid",
    "content": "请解释Java中的多态性",
    "type": "technical",
    "difficulty": "medium"
  }
}

// 答案回传
{
  "action": "submit_answer",
  "session_id": "uuid",
  "question_id": "q_uuid",
  "answer": {
    "text": "用户回答内容",
    "audio_url": "oss://path/to/audio",
    "timestamp": "2026-02-13T10:30:00Z"
  }
}
```

### 4.2 内部模块通信
- 同步调用: 直接函数调用
- 异步任务: Celery + Redis (用于耗时操作如向量化、报告生成)

## 5. 数据流设计

### 5.1 面试流程数据流

```
用户发起面试
    ↓
主服务创建会话 → 存储到PostgreSQL
    ↓
调用AI数字人微服务启动面试
    ↓
AI数字人微服务请求题目 → 主服务题库模块
    ↓
题库模块基于岗位+RAG检索返回题目
    ↓
AI数字人进行对话交互
    ↓
用户回答 → AI数字人收集 → 回传主服务
    ↓
主服务评估模块分析答案
    ↓
评估结果存储 + 实时反馈
    ↓
面试结束 → 生成综合报告
    ↓
能力提升模块生成个性化建议
```

### 5.2 知识库RAG流程

```
管理员上传文档
    ↓
文档解析(PDF/Word/Markdown)
    ↓
文本分块(Chunking)
    ↓
向量化(Embeddings)
    ↓
存储到Chroma DB + 元数据存储到PostgreSQL
    ↓
面试时检索:
  用户问题 → 向量化 → Chroma相似度搜索
    ↓
  返回Top-K相关文档片段
    ↓
  结合LLM生成答案或评估标准
```

## 6. 部署架构

### 6.1 开发环境
```
Docker Compose 一键部署:
- FastAPI主服务容器
- AI数字人微服务容器
- PostgreSQL容器
- Redis容器
- Chroma DB容器
```

### 6.2 生产环境
```
Kubernetes集群部署:
- 主服务: 多副本 + HPA自动扩缩容
- AI数字人微服务: 独立部署 + GPU节点支持
- 数据库: 云数据库RDS (PostgreSQL)
- 缓存: 云Redis集群
- 向量数据库: Chroma独立部署或云向量数据库
- 对象存储: MinIO集群 / 阿里云OSS
- 负载均衡: Nginx Ingress / 云负载均衡
```

### 6.3 扩展性考虑
- **水平扩展**: 主服务无状态设计,支持多实例部署
- **垂直扩展**: AI微服务支持GPU加速
- **数据库分片**: 用户数据按user_id分片
- **缓存策略**: 多级缓存(本地缓存 + Redis)
- **CDN加速**: 静态资源和音视频文件

## 7. 安全设计

### 7.1 认证安全
- JWT Token + Refresh Token机制
- Token过期时间: Access Token 2小时, Refresh Token 7天
- 密码加密: bcrypt算法
- 防暴力破解: 登录失败次数限制

### 7.2 API安全
- HTTPS强制加密
- API限流: 基于用户/IP的请求频率限制
- CORS跨域配置
- SQL注入防护: ORM参数化查询
- XSS防护: 输入验证与输出转义

### 7.3 数据安全
- 敏感数据加密存储
- 数据库访问权限最小化
- 定期备份策略
- 审计日志记录

## 8. 监控与日志

### 8.1 监控指标
- 服务健康检查: /health端点
