## 1. 概述

### 1.1 模块说明

题库模块包含两层结构，并支持社区功能：
- **题库（Question Bank）**：题目的容器，按主题或岗位分类，**每个题库归属于创建者**
- **题目（Question）**：具体的面试题，包含题目内容、参考答案、评分标准等
- **社区**：用户可将自己的题库申请上架社区，经管理员审核后对所有用户可见；其他用户可复制社区题库到自己名下

### 1.2 API 前缀

```
/api/v1/questions
```

### 1.3 认证要求

所有接口均需要登录（`Authorization: Bearer <access_token>`），管理员接口额外要求 `role == "admin"`。

| 接口 | 认证要求 |
|------|---------|
| 获取我的题库列表 | ✅ 登录 |
| 创建/获取/更新/删除题库 | ✅ 登录（仅自己的） |
| 浏览社区题库 | ✅ 登录 |
| 申请上架社区 | ✅ 登录（仅自己的） |
| 审核社区申请 / 查看待审核 | ✅ **管理员** |
| 复制社区题库 | ✅ 登录 |
| 题目 CRUD | ✅ 登录（校验题库归属/访问权限） |

### 1.4 社区状态枚举（community_status）

| 值 | 说明 |
|-----|------|
| `null` | 私有（默认） |
| `pending` | 待审核 |
| `approved` | 已上架社区 |
| `rejected` | 已拒绝（可重新申请） |

### 1.5 核心业务规则

1. **用户隔离**：每个用户只能看到/操作自己创建的题库
2. **社区公开**：`community_status == "approved"` 的题库对所有登录用户可见
3. **深拷贝**：复制社区题库时，创建全新的题库 + 全新的题目，与原题库完全独立
4. **自动下架**：用户修改已上架（approved）的题库时，自动将其从社区下架（`community_status → null`）
5. **管理员最高权限**：可批准/拒绝/强制下架任何社区题库

### 1.6 题目类型枚举

| 值 | 说明 |
|-----|------|
| `technical` | 技术题 |
| `scenario` | 场景题 |
| `algorithm` | 算法题 |
| `behavioral` | 行为面试题 |

### 1.7 难度等级枚举

| 值 | 说明 |
|-----|------|
| `easy` | 简单 |
| `medium` | 中等 |
| `hard` | 困难 |

## 2. 个人题库 API

### 2.1 获取我的题库列表

**GET** `/api/v1/questions/banks`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | ❌ | 页码，默认 1 |
| page_size | int | ❌ | 每页数量，默认 20，范围 1-100 |
| category | string | ❌ | 分类筛选 |
| status | string | ❌ | 状态筛选 |

> 仅返回当前用户创建的题库。

**响应示例：**

```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "items": [
      {
        "id": "uuid",
        "name": "前端基础题库",
        "description": "HTML/CSS/JS 面试题",
        "category": "frontend",
        "tags": ["HTML", "CSS"],
        "status": "active",
        "community_status": null,
        "created_by": "current-user-id",
        "created_at": "2026-02-14T10:00:00Z"
      }
    ],
    "total": 15,
    "page": 1,
    "page_size": 20,
    "total_pages": 1
  }
}
```

---

### 2.2 创建题库

**POST** `/api/v1/questions/banks`

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | ✅ | 题库名称，最大 100 字符 |
| description | string | ❌ | 题库描述 |
| category | string | ❌ | 分类 |
| tags | string[] | ❌ | 标签列表 |

**响应：** 返回创建的题库对象（含 `community_status: null`）。

---

### 2.3 获取题库详情

**GET** `/api/v1/questions/banks/{bank_id}`

> 仅能获取自己创建的题库。

| HTTP 状态码 | 错误信息 |
|------------|---------|
| 404 | 题库不存在 |

---

### 2.4 更新题库

**PUT** `/api/v1/questions/banks/{bank_id}`

> 仅能更新自己创建的题库。**若题库已上架社区（approved），更新后将自动下架。**

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | ❌ | 题库名称 |
| description | string | ❌ | 题库描述 |
| category | string | ❌ | 分类 |
| tags | string[] | ❌ | 标签列表 |
| status | string | ❌ | 状态 |

---

### 2.5 删除题库

**DELETE** `/api/v1/questions/banks/{bank_id}`

> 仅能删除自己创建的题库（软删除）。

## 3. 社区题库 API

### 3.1 浏览社区题库

**GET** `/api/v1/questions/community/banks`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | ❌ | 页码，默认 1 |
| page_size | int | ❌ | 每页数量，默认 20 |
| category | string | ❌ | 分类筛选 |

> 仅返回 `community_status == "approved"` 的题库。

**响应：** 与题库列表格式相同，所有题库的 `community_status` 为 `"approved"`。

---

### 3.2 申请上架社区

**POST** `/api/v1/questions/banks/{bank_id}/apply-community`

> 将自己的题库申请上架社区。`null` 或 `rejected` 状态可申请（→ `pending`）。已 `pending` 或 `approved` 状态则直接返回。

**响应示例：**

```json
{
  "code": 200,
  "message": "申请已提交",
  "data": {
    "id": "bank-uuid",
    "name": "React 进阶题库",
    "community_status": "pending",
    "..."
  }
}
```

---

### 3.3 管理员审核社区申请

**POST** `/api/v1/questions/banks/{bank_id}/review-community`

> ⚠️ 需要管理员权限（`role == "admin"`）

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | string | ✅ | `approve`（批准）/ `reject`（拒绝）/ `delist`（强制下架） |

**业务规则：**
- `approve`：`pending` → `approved`
- `reject`：`pending` → `rejected`
- `delist`：`approved` → `null`（强制下架）

| HTTP 状态码 | 错误信息 |
|------------|---------|
| 400 | 无效的操作 |
| 403 | 需要管理员权限 |
| 404 | 题库不存在 |

---

### 3.4 管理员查看待审核列表

**GET** `/api/v1/questions/community/pending`

> ⚠️ 需要管理员权限

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | ❌ | 页码，默认 1 |
| page_size | int | ❌ | 每页数量，默认 20 |

**响应：** 分页返回所有 `community_status == "pending"` 的题库。

---

### 3.5 复制社区题库

**POST** `/api/v1/questions/community/banks/{bank_id}/copy`

> 深拷贝一个社区题库（含所有题目）到当前用户名下。新题库 `community_status` 为 `null`，修改不影响原题库。

**响应：** 返回新创建的题库对象。

| HTTP 状态码 | 错误信息 |
|------------|---------|
| 404 | 社区题库不存在 |

## 4. 题目 API

### 4.1 获取题目列表

**GET** `/api/v1/questions`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bank_id | string | ✅ | 所属题库 ID（**必填**） |
| page | int | ❌ | 页码，默认 1 |
| page_size | int | ❌ | 每页数量，默认 20 |
| type | string | ❌ | 题目类型 |
| category | string | ❌ | 分类筛选 |
| difficulty | string | ❌ | 难度筛选 |
| status | string | ❌ | 状态筛选 |

> **访问权限**：`bank_id` 对应的题库必须是当前用户创建的 **或** `community_status == "approved"`。

| HTTP 状态码 | 错误信息 |
|------------|---------|
| 404 | 题库不存在或无权访问 |

---

### 4.2 创建题目

**POST** `/api/v1/questions`

> 校验 `bank_id` 对应题库必须为当前用户所有。

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bank_id | string | ✅ | 所属题库 ID |
| title | string | ✅ | 题目标题，最大 200 字符 |
| content | string | ✅ | 题目内容 |
| type | string | ✅ | 题目类型 |
| category | string | ❌ | 分类 |
| difficulty | string | ✅ | 难度 |
| difficulty_score | int | ❌ | 难度分数 1-10 |
| tags | string[] | ❌ | 标签列表 |
| reference_answer | string | ❌ | 参考答案 |
| answer_key_points | string[] | ❌ | 答案要点 |
| scoring_criteria | object | ❌ | 评分标准 |

| HTTP 状态码 | 错误信息 |
|------------|---------|
| 400 | 所属题库不存在或无权操作 |

---

### 4.3 批量创建题目

**POST** `/api/v1/questions/batch`

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| bank_id | string | ✅ | 所属题库 ID |
| questions | QuestionItem[] | ✅ | 题目数组，1-100 条 |

> `QuestionItem` 字段同创建题目（不含 `bank_id`）。

---

### 4.4 获取题目详情

**GET** `/api/v1/questions/{question_id}`

> 校验题目所属题库的访问权限（当前用户的 或 approved 的社区题库）。

返回包含参考答案、答案要点、评分标准的完整题目。

---

### 4.5 更新题目

**PUT** `/api/v1/questions/{question_id}`

> 校验题目所属题库必须为当前用户所有。

| HTTP 状态码 | 错误信息 |
|------------|---------|
| 403 | 无权操作该题目 |
| 404 | 题目不存在 |

---

### 4.6 删除题目

**DELETE** `/api/v1/questions/{question_id}`

> 校验题目所属题库必须为当前用户所有。

## 5. TypeScript 类型定义

创建 `src/types/question.ts`：

```typescript
// ============ 题库类型 ============

/** 社区状态 */
export type CommunityStatus = null | 'pending' | 'approved' | 'rejected'

/** 创建题库请求 */
export interface QuestionBankCreate {
  name: string
  description?: string
  category?: string
  tags?: string[]
}

/** 更新题库请求 */
export interface QuestionBankUpdate {
  name?: string
  description?: string
  category?: string
  tags?: string[]
  status?: 'active' | 'inactive'
}

/** 题库响应 */
export interface QuestionBank {
  id: string
  name: string
  description: string | null
  category: string | null
  tags: string[] | null
  status: string
  community_status: CommunityStatus
  created_by: string | null
  created_at: string
}

/** 题库列表查询参数 */
export interface QuestionBankListParams {
  page?: number
  page_size?: number
  category?: string
  status?: string
}

/** 社区审核请求 */
export interface CommunityReviewRequest {
  action: 'approve' | 'reject' | 'delist'
}

// ============ 题目类型 ============

export type QuestionType = 'technical' | 'scenario' | 'algorithm' | 'behavioral'
export type Difficulty = 'easy' | 'medium' | 'hard'

/** 创建题目请求 */
export interface QuestionCreate {
  bank_id: string
  title: string
  content: string
  type: QuestionType
  category?: string
  difficulty: Difficulty
  difficulty_score?: number
  tags?: string[]
  reference_answer?: string
  answer_key_points?: string[]
  scoring_criteria?: Record<string, number>
}

/** 批量创建题目请求 */
export interface QuestionBatchCreate {
  bank_id: string
  questions: Omit<QuestionCreate, 'bank_id'>[]
}

/** 更新题目请求 */
export interface QuestionUpdate {
  title?: string
  content?: string
  type?: QuestionType
  category?: string
  difficulty?: Difficulty
  difficulty_score?: number
  tags?: string[]
  reference_answer?: string
  answer_key_points?: string[]
  scoring_criteria?: Record<string, number>
  status?: 'active' | 'inactive'
}

/** 题目列表响应 */
export interface Question {
  id: string
  bank_id: string
  title: string
  content: string
  type: QuestionType
  category: string | null
  difficulty: Difficulty
  difficulty_score: number | null
  tags: string[] | null
  usage_count: number
  status: string
  created_at: string
}

/** 题目详情响应（包含答案） */
export interface QuestionDetail extends Question {
  reference_answer: string | null
  answer_key_points: string[] | null
  scoring_criteria: Record<string, number> | null
}

/** 题目列表查询参数 */
export interface QuestionListParams {
  bank_id: string  // 必填
  page?: number
  page_size?: number
  type?: QuestionType
  category?: string
  difficulty?: Difficulty
  status?: string
}

/** 批量创建题目响应 */
export interface QuestionBatchCreateResponse {
  total: number
  questions: Question[]
}

// ============ 通用类型 ============

export interface PaginatedResponse<T> {
  items: T[]
  total: number
  page: number
  page_size: number
  total_pages: number
}

export interface ApiResponse<T = any> {
  code: number
  message: string
  data: T
}
```

## 6. API 封装实现

创建 `src/api/question.ts`：

```typescript
import request from '@/utils/request'
import type {
  QuestionBankCreate, QuestionBankUpdate, QuestionBankListParams, QuestionBank,
  CommunityReviewRequest,
  QuestionCreate, QuestionBatchCreate, QuestionUpdate, QuestionListParams,
  Question, QuestionDetail, QuestionBatchCreateResponse,
  PaginatedResponse, ApiResponse
} from '@/types/question'

const PREFIX = '/questions'

// ============ 个人题库 ============

export const listMyBanks = (params?: QuestionBankListParams): Promise<ApiResponse<PaginatedResponse<QuestionBank>>> =>
  request.get(`${PREFIX}/banks`, { params })

export const createBank = (data: QuestionBankCreate): Promise<ApiResponse<QuestionBank>> =>
  request.post(`${PREFIX}/banks`, data)

export const getBank = (bankId: string): Promise<ApiResponse<QuestionBank>> =>
  request.get(`${PREFIX}/banks/${bankId}`)

export const updateBank = (bankId: string, data: QuestionBankUpdate): Promise<ApiResponse<QuestionBank>> =>
  request.put(`${PREFIX}/banks/${bankId}`, data)

export const deleteBank = (bankId: string): Promise<ApiResponse<null>> =>
  request.delete(`${PREFIX}/banks/${bankId}`)

// ============ 社区题库 ============

export const listCommunityBanks = (params?: Omit<QuestionBankListParams, 'status'>): Promise<ApiResponse<PaginatedResponse<QuestionBank>>> =>
  request.get(`${PREFIX}/community/banks`, { params })

export const applyCommunity = (bankId: string): Promise<ApiResponse<QuestionBank>> =>
  request.post(`${PREFIX}/banks/${bankId}/apply-community`)

export const reviewCommunity = (bankId: string, data: CommunityReviewRequest): Promise<ApiResponse<QuestionBank>> =>
  request.post(`${PREFIX}/banks/${bankId}/review-community`, data)

export const listPendingBanks = (params?: { page?: number; page_size?: number }): Promise<ApiResponse<PaginatedResponse<QuestionBank>>> =>
  request.get(`${PREFIX}/community/pending`, { params })

export const copyCommunityBank = (bankId: string): Promise<ApiResponse<QuestionBank>> =>
  request.post(`${PREFIX}/community/banks/${bankId}/copy`)

// ============ 题目 ============

export const listQuestions = (params: QuestionListParams): Promise<ApiResponse<PaginatedResponse<Question>>> =>
  request.get(PREFIX, { params })

export const createQuestion = (data: QuestionCreate): Promise<ApiResponse<Question>> =>
  request.post(PREFIX, data)

export const batchCreateQuestions = (data: QuestionBatchCreate): Promise<ApiResponse<QuestionBatchCreateResponse>> =>
  request.post(`${PREFIX}/batch`, data)

export const getQuestion = (questionId: string): Promise<ApiResponse<QuestionDetail>> =>
  request.get(`${PREFIX}/${questionId}`)

export const updateQuestion = (questionId: string, data: QuestionUpdate): Promise<ApiResponse<Question>> =>
  request.put(`${PREFIX}/${questionId}`, data)

export const deleteQuestion = (questionId: string): Promise<ApiResponse<null>> =>
  request.delete(`${PREFIX}/${questionId}`)
```

## 7. 错误处理

### 7.1 错误码对照表

| HTTP 状态码 | 业务含义 | 前端处理建议 |
|------------|---------|-------------|
| 200 | 请求成功 | 正常处理响应数据 |
| 400 | 请求参数错误 / 题库不存在 | 显示错误提示 |
| 401 | 未认证 / Token 过期 | 清除登录状态，跳转登录页 |
| 403 | 权限不足 / 需要管理员权限 / 无权操作该题目 | 显示无权限提示 |
| 404 | 资源不存在或无权访问 | 显示 404 或提示 |
| 422 | 参数验证失败 | 显示具体验证错误 |

### 7.2 常见错误场景

| 场景 | 错误信息 | 说明 |
|------|---------|------|
| 操作他人题库 | 题库不存在 | 归属校验失败统一返回 404 |
| 操作他人题目 | 无权操作该题目 | 返回 403 |
| 非管理员审核 | 需要管理员权限 | 返回 403 |
| 无效审核操作 | 无效的操作 | action 参数不合法 |
| 复制非社区题库 | 社区题库不存在 | 仅 approved 可复制 |
| 访问无权题目列表 | 题库不存在或无权访问 | bank_id 非自己的且非社区的 |
