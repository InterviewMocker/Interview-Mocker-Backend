## 2. 数据库系统设计

### 2.1 开发阶段：SQLite

两个服务共享同一个 SQLite 数据库文件：

```
data/interview_mocker.db
```

**配置方式：**
```python
# .env
DATABASE_URL=sqlite+aiosqlite:///./data/interview_mocker.db
```

### 2.2 生产阶段：PostgreSQL

```python
# .env.production
DATABASE_URL=postgresql+asyncpg://user:pass@localhost:5432/interview_db
```

### 2.3 共享数据库连接

```python
# shared/database/connection.py

import os
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine, async_sessionmaker
from shared.config.settings import get_shared_settings

settings = get_shared_settings()

# 确保数据目录存在
os.makedirs("./data", exist_ok=True)

# 创建共享引擎 - 两个服务使用相同的数据库
engine = create_async_engine(
    settings.DATABASE_URL,
    echo=settings.DEBUG,
    future=True,
)

AsyncSessionLocal = async_sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,
)


async def get_db_session():
    """获取数据库会话"""
    async with AsyncSessionLocal() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise
```

## 3. 数据库迁移同步方案

### 3.1 使用 Alembic 管理迁移

迁移脚本放在项目根目录的 `migrations/` 文件夹，由两个团队共同维护。

**初始化 Alembic：**
```bash
# 在项目根目录执行
alembic init migrations
```

**配置 `migrations/env.py`：**
```python
# migrations/env.py

from logging.config import fileConfig
from sqlalchemy import pool
from sqlalchemy.engine import Connection
from alembic import context

# 导入共享模型
from shared.models import Base
from shared.config.settings import get_shared_settings

settings = get_shared_settings()
config = context.config

# 设置数据库 URL
config.set_main_option("sqlalchemy.url", settings.DATABASE_URL.replace("+aiosqlite", ""))

if config.config_file_name is not None:
    fileConfig(config.config_file_name)

target_metadata = Base.metadata


def run_migrations_offline() -> None:
    """离线迁移"""
    url = config.get_main_option("sqlalchemy.url")
    context.configure(
        url=url,
        target_metadata=target_metadata,
        literal_binds=True,
        dialect_opts={"paramstyle": "named"},
    )
    with context.begin_transaction():
        context.run_migrations()


def run_migrations_online() -> None:
    """在线迁移"""
    connectable = engine_from_config(
        config.get_section(config.config_ini_section, {}),
        prefix="sqlalchemy.",
        poolclass=pool.NullPool,
    )
    with connectable.connect() as connection:
        context.configure(
            connection=connection, 
            target_metadata=target_metadata
        )
        with context.begin_transaction():
            context.run_migrations()


if context.is_offline_mode():
    run_migrations_offline()
else:
    run_migrations_online()
```

### 3.2 迁移工作流程

**团队 A 或 B 修改数据模型后：**

```bash
# 1. 在 shared/models/ 中修改模型文件

# 2. 生成迁移脚本
alembic revision --autogenerate -m "描述变更内容"

# 3. 检查生成的迁移脚本 (migrations/versions/xxx.py)

# 4. 应用迁移
alembic upgrade head

# 5. 提交代码 (包含 shared/models 和 migrations/versions)
git add shared/models/ migrations/versions/
git commit -m "feat: 添加xxx字段"
git push
```

**另一个团队拉取代码后：**

```bash
# 1. 拉取最新代码
git pull

# 2. 应用迁移
alembic upgrade head
```

### 3.3 迁移冲突解决

当两个团队同时修改模型时，可能产生迁移冲突：

```bash
# 1. 查看当前版本
alembic current

# 2. 查看迁移历史
alembic history

# 3. 如果有分支冲突，合并迁移
alembic merge -m "merge migrations" revision1 revision2

# 4. 重新应用
alembic upgrade head
```

### 3.4 快捷脚本

```python
# scripts/migrate.py

import subprocess
import sys

def run_command(cmd):
    result = subprocess.run(cmd, shell=True)
    return result.returncode == 0

def migrate():
    """运行数据库迁移"""
    commands = {
        "upgrade": "alembic upgrade head",
        "downgrade": "alembic downgrade -1",
        "current": "alembic current",
        "history": "alembic history",
        "generate": lambda msg: f'alembic revision --autogenerate -m "{msg}"'
    }
    
    if len(sys.argv) < 2:
        print("Usage: python scripts/migrate.py <command> [message]")
        print("Commands: upgrade, downgrade, current, history, generate")
        return
    
    cmd = sys.argv[1]
    
    if cmd == "generate":
        if len(sys.argv) < 3:
            print("Please provide a migration message")
            return
        run_command(commands["generate"](sys.argv[2]))
    elif cmd in commands:
        run_command(commands[cmd])
    else:
        print(f"Unknown command: {cmd}")

if __name__ == "__main__":
    migrate()
```

│                     │  ◄────────────────── │                     │

### 4.3 HTTP 客户端

```python
# interview_avatar_service/clients/main_service.py

import httpx
from shared.config.settings import get_shared_settings

settings = get_shared_settings()


class MainServiceClient:
    """调用主服务的 HTTP 客户端"""
    
    def __init__(self):
        self.base_url = settings.MAIN_SERVICE_URL
        self.headers = {
            "X-Service-Token": settings.INTERNAL_SERVICE_TOKEN,
            "Content-Type": "application/json"
        }
    
    async def get_next_question(self, session_id: str, context: dict):
        """获取下一个问题"""
        async with httpx.AsyncClient() as client:
            response = await client.post(
                f"{self.base_url}/api/v1/questions/next",
                headers=self.headers,
                json={"session_id": session_id, "context": context}
            )
            response.raise_for_status()
            return response.json()["data"]
    
    async def submit_answer(self, session_id: str, question_id: str, answer: dict):
        """提交用户答案"""
        async with httpx.AsyncClient() as client:
            response = await client.post(
                f"{self.base_url}/api/v1/interviews/answers",
                headers=self.headers,
                json={
                    "session_id": session_id,
                    "question_id": question_id,
                    "answer": answer
                }
            )
            response.raise_for_status()
            return response.json()["data"]
```

## 5. 两个团队的开发流程

### 5.1 初始设置（两个团队都需要）

```bash
# 1. 克隆代码
git clone <repo_url>
cd Interview-Mocker-Backend

# 2. 创建虚拟环境
uv sync

# 3. 配置环境变量
cp .env.example .env

# 4. 初始化数据库
uv run alembic upgrade head
```

### 5.2 团队 A：主服务开发

```bash
# 启动主服务
uv run uvicorn main_service.main:app --reload --port 8000

# 访问 API 文档
# http://localhost:8000/docs
```

**开发职责：**
- `main_service/` 目录下的所有代码
- `shared/models/` 数据模型（需与团队B协调）
- `migrations/` 迁移脚本

### 5.3 团队 B：数字人服务开发

```bash
# 启动数字人服务
uv run uvicorn interview_avatar_service.main:app --reload --port 8001

# 访问 API 文档
# http://localhost:8001/docs
```

**开发职责：**
- `interview_avatar_service/` 目录下的所有代码
- 调用主服务的 HTTP 客户端
- 数字人渲染、语音处理等

### 5.4 模型变更协调流程

```
团队A 或 团队B 需要修改数据模型
         │
         ▼
  在 shared/models/ 中修改
         │
         ▼
  生成迁移脚本: alembic revision --autogenerate -m "xxx"
         │
         ▼
  提交 PR，另一个团队 Review
         │
         ▼
  合并后，两个团队都执行: alembic upgrade head
```

### 6.3 方法三：启动脚本

```python
# scripts/start_services.py

import subprocess
import sys
import time
import signal

processes = []


def signal_handler(sig, frame):
    print("\nStopping services...")
    for p in processes:
        p.terminate()
    sys.exit(0)


def main():
    signal.signal(signal.SIGINT, signal_handler)
    
    print("Starting Main Service on port 8000...")
    main_service = subprocess.Popen([
        sys.executable, "-m", "uvicorn",
        "main_service.main:app",
        "--host", "0.0.0.0",
        "--port", "8000",
        "--reload"
    ])
    processes.append(main_service)
    
    time.sleep(2)  # 等待主服务启动
    
    print("Starting Avatar Service on port 8001...")
    avatar_service = subprocess.Popen([
        sys.executable, "-m", "uvicorn",
        "interview_avatar_service.main:app",
        "--host", "0.0.0.0",
        "--port", "8001",
        "--reload"
    ])
    processes.append(avatar_service)
    
    print("\n=== Services Started ===")
    print("Main Service:   http://localhost:8000/docs")
    print("Avatar Service: http://localhost:8001/docs")
    print("\nPress Ctrl+C to stop all services\n")
    
    # 等待进程
    for p in processes:
        p.wait()


if __name__ == "__main__":
    main()
```
