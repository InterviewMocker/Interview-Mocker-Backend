### 数据库访问

#### 方式一：使用共享数据库连接

```python
# main_service/core/database.py
from shared.database import get_db_session, init_shared_db, close_shared_db

# 直接使用共享的数据库依赖
get_db = get_db_session
```

#### 方式二：在路由中使用

```python
from fastapi import Depends
from sqlalchemy.ext.asyncio import AsyncSession
from shared.database import get_db_session
from shared.models import User

@router.get("/users/{user_id}")
async def get_user(user_id: str, db: AsyncSession = Depends(get_db_session)):
    result = await db.execute(
        select(User).where(User.id == user_id)
    )
    user = result.scalar_one_or_none()
    return user
```

#### 方式三：在服务层中使用

```python
# main_service/services/user_service.py
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession
from shared.models import User

class UserService:
    def __init__(self, db: AsyncSession):
        self.db = db
    
    async def get_by_id(self, user_id: str) -> User | None:
        result = await self.db.execute(
            select(User).where(User.id == user_id)
        )
        return result.scalar_one_or_none()
    
    async def create(self, username: str, email: str) -> User:
        user = User(username=username, email=email)
        self.db.add(user)
        await self.db.commit()
        await self.db.refresh(user)
        return user
```

### 数据库访问

数字人服务与主服务**共享同一个数据库**，使用相同的方式访问：

#### 读取数据

```python
from fastapi import Depends
from sqlalchemy.ext.asyncio import AsyncSession
from shared.database import get_db_session
from shared.models import InterviewSession, InterviewQARecord

@router.get("/sessions/{session_id}")
async def get_session(
    session_id: str,
    db: AsyncSession = Depends(get_db_session)
):
    result = await db.execute(
        select(InterviewSession).where(InterviewSession.id == session_id)
    )
    return result.scalar_one_or_none()
```

#### 写入数据

```python
from shared.models import InterviewQARecord

async def save_qa_record(
    db: AsyncSession,
    session_id: str,
    question_id: str,
    answer_text: str
):
    record = InterviewQARecord(
        session_id=session_id,
        question_id=question_id,
        answer_text=answer_text
    )
    db.add(record)
    await db.commit()
    await db.refresh(record)
    return record
```

### 冲突解决

如果两个团队同时生成了迁移脚本：

```bash
# 发现冲突后
git pull

# 如果有迁移冲突，手动解决
# 1. 查看 migrations/versions/ 中的文件
# 2. 确保 down_revision 链接正确
# 3. 可能需要调整其中一个迁移的 down_revision

# 重新应用迁移
uv run python scripts/migrate.py upgrade
```

### 最佳实践

1. **模型变更前沟通** - 在修改 `shared/models/` 前，两个团队应先沟通
2. **频繁同步** - 定期 `git pull` 并 `migrate.py upgrade`
3. **小步提交** - 每次只做一个模型变更
4. **迁移描述清晰** - 使用有意义的迁移消息

### Q: 数据库文件在哪里？

```
data/
├── interview_mocker.db    # SQLite 数据库
└── chroma/                # ChromaDB 向量数据库
```

```bash
# 删除数据库文件
rm data/interview_mocker.db

# 重新初始化
uv run python scripts/migrate.py init
```

### Q: 如何查看迁移历史？

```bash
uv run python scripts/migrate.py history
```

### Q: 主服务和数字人服务可以同时写同一张表吗？

可以，但需要注意：
- SQLite 支持多进程读写（通过 WAL 模式）
- 生产环境使用 PostgreSQL，天然支持并发
- 避免同时更新同一条记录

### Q: 如何测试服务间通信？

```bash
# 1. 启动主服务
uv run uvicorn main_service.main:app --port 8000

# 2. 启动数字人服务
uv run uvicorn interview_avatar_service.main:app --port 8001

# 3. 测试健康检查
curl http://localhost:8001/health
# 返回：{"service": "avatar-service", "checks": {"main_service": "reachable"}}
```
