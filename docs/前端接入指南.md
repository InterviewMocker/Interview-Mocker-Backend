## 2. 环境配置

### 2.1 创建环境变量文件

在前端项目根目录创建以下文件：

**`.env.development`** - 开发环境：

```env
# 后端服务地址
VITE_API_BASE_URL=http://localhost:8000
VITE_API_PREFIX=/api/v1

# 数字人服务地址（如需直接调用）
VITE_AVATAR_API_URL=http://localhost:8001
VITE_AVATAR_API_PREFIX=/api/v1

# 开发模式
VITE_APP_ENV=development
```

**`.env.production`** - 生产环境：

```env
# 生产环境后端地址（根据实际部署修改）
VITE_API_BASE_URL=https://api.interview-mocker.com
VITE_API_PREFIX=/api/v1

VITE_AVATAR_API_URL=https://avatar.interview-mocker.com
VITE_AVATAR_API_PREFIX=/api/v1

VITE_APP_ENV=production
```

**`.env.staging`** - 测试环境：

```env
VITE_API_BASE_URL=https://staging-api.interview-mocker.com
VITE_API_PREFIX=/api/v1

VITE_AVATAR_API_URL=https://staging-avatar.interview-mocker.com
VITE_AVATAR_API_PREFIX=/api/v1

VITE_APP_ENV=staging
```

### 2.2 TypeScript 环境变量类型声明

创建 `src/env.d.ts`：

```typescript
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_BASE_URL: string
  readonly VITE_API_PREFIX: string
  readonly VITE_AVATAR_API_URL: string
  readonly VITE_AVATAR_API_PREFIX: string
  readonly VITE_APP_ENV: 'development' | 'staging' | 'production'
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
```

### 2.3 Vite 配置代理（可选）

如果遇到跨域问题，可在 `vite.config.ts` 中配置代理：

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 5173,
    proxy: {
      // 主服务代理
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
        secure: false
      },
      // 数字人服务代理
      '/avatar-api': {
        target: 'http://localhost:8001',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/avatar-api/, '/api')
      }
    }
  }
})
```

使用代理时，环境变量改为：

```env
# 使用代理时的配置
VITE_API_BASE_URL=
VITE_API_PREFIX=/api/v1
```

## 3. 跨域配置

### 3.1 后端已配置 CORS

后端服务已配置跨域支持，允许以下来源：

- `http://localhost:5173` (Vite 默认端口)
- `http://localhost:3000`
- `http://127.0.0.1:5173`

如需添加其他来源，修改后端 `main_service/main.py`：

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:5173",
        "http://localhost:3000",
        # 添加您的前端地址
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 3.2 前端请求配置

确保 Axios 请求携带凭证（如果使用 Cookie）：

```typescript
const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL + import.meta.env.VITE_API_PREFIX,
  timeout: 10000,
  withCredentials: true,  // 允许携带凭证
  headers: {
    'Content-Type': 'application/json'
  }
})
```

## 4. 连接测试

### 4.1 启动后端服务

```bash
# 在后端项目目录
cd Interview-Mocker-Backend

# 同时启动两个服务
uv run python run.py

# 或仅启动主服务
uv run python run.py main
```

### 4.2 验证后端服务

```bash
# 检查主服务健康状态
curl http://localhost:8000/health

# 检查数字人服务健康状态
curl http://localhost:8001/health

# 访问 API 文档
# 主服务: http://localhost:8000/docs
# 数字人服务: http://localhost:8001/docs
```

### 4.3 前端连接测试

创建 `src/utils/healthCheck.ts`：

```typescript
import axios from 'axios'

interface HealthStatus {
  main: boolean
  avatar: boolean
  mainVersion?: string
  avatarVersion?: string
}

export async function checkBackendHealth(): Promise<HealthStatus> {
  const status: HealthStatus = {
    main: false,
    avatar: false
  }

  // 检查主服务
  try {
    const mainRes = await axios.get(
      `${import.meta.env.VITE_API_BASE_URL}/health`,
      { timeout: 3000 }
    )
    status.main = mainRes.status === 200
    status.mainVersion = mainRes.data?.version
  } catch {
    status.main = false
  }

  // 检查数字人服务
  try {
    const avatarRes = await axios.get(
      `${import.meta.env.VITE_AVATAR_API_URL}/health`,
      { timeout: 3000 }
    )
    status.avatar = avatarRes.status === 200
    status.avatarVersion = avatarRes.data?.version
  } catch {
    status.avatar = false
  }

  return status
}
```

在 `App.vue` 中使用：

```vue
<script setup lang="ts">
import { onMounted, ref } from 'vue'
import { checkBackendHealth } from '@/utils/healthCheck'

const backendStatus = ref({ main: false, avatar: false })

onMounted(async () => {
  if (import.meta.env.DEV) {
    backendStatus.value = await checkBackendHealth()
    
    if (!backendStatus.value.main) {
      console.warn('⚠️ 主服务未连接，请确保后端服务已启动')
      console.warn('   启动命令: uv run python run.py')
    } else {
      console.log('✅ 主服务已连接')
    }
  }
})
</script>
```

**步骤 3：访问应用**

- 前端应用：`http://localhost:5173`
- 后端 API 文档：`http://localhost:8000/docs`

### 5.2 API 调用流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        前端调用 API 流程                             │
└─────────────────────────────────────────────────────────────────────┘

1. 用户操作 (如点击登录按钮)
         │
         ▼
2. Vue 组件调用 API 函数
   login({ username: 'xxx', password: 'xxx' })
         │
         ▼
3. Axios 请求拦截器
   - 自动添加 Authorization 头 (如果已登录)
   - 添加 Content-Type
         │
         ▼
4. 发送 HTTP 请求
   POST http://localhost:8000/api/v1/auth/login
         │
         ▼
5. 后端处理请求并返回响应
   { "code": 200, "message": "登录成功", "data": {...} }
         │
         ▼
6. Axios 响应拦截器
   - 统一处理错误 (401/403/500等)
   - 提取 response.data
         │
         ▼
7. 组件接收数据并更新状态
   - 保存 Token 到 localStorage
   - 更新 Pinia Store
   - 跳转到首页
```

## 6. 常见问题

### Q1: 请求报错 "Network Error"

**原因**：后端服务未启动或地址配置错误

**解决方案**：
```bash
# 1. 确认后端服务已启动
curl http://localhost:8000/health

# 2. 检查环境变量配置
cat .env.development

# 3. 启动后端服务
cd Interview-Mocker-Backend
uv run python run.py
```

### Q2: 请求报错 "CORS policy"

**原因**：跨域请求被阻止

**解决方案**：
1. 确认后端 CORS 配置包含前端地址
2. 或使用 Vite 代理（见 2.3 节）

### Q3: 登录后请求仍返回 401

**原因**：Token 未正确携带

**解决方案**：
```typescript
// 检查请求拦截器是否正确添加 Token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### Q4: 如何切换到生产环境

```bash
# 构建生产版本
pnpm build

# 预览生产版本
pnpm preview

# 部署到服务器后，确保环境变量指向生产后端
```

### Q5: 如何调试 API 请求

```typescript
// 在 request.ts 中添加日志
request.interceptors.request.use((config) => {
  if (import.meta.env.DEV) {
    console.log('🚀 Request:', config.method?.toUpperCase(), config.url)
    console.log('   Headers:', config.headers)
    console.log('   Data:', config.data)
  }
  return config
})

request.interceptors.response.use(
  (response) => {
    if (import.meta.env.DEV) {
      console.log('✅ Response:', response.config.url, response.data)
    }
    return response.data
  },
  (error) => {
    if (import.meta.env.DEV) {
      console.error('❌ Error:', error.config?.url, error.response?.data)
    }
    return Promise.reject(error)
  }
)
```

### Q6: WebSocket 连接（数字人服务）

如果数字人服务使用 WebSocket 进行实时通信：

```typescript
// src/utils/websocket.ts
export function createAvatarWebSocket(sessionId: string): WebSocket {
  const wsUrl = import.meta.env.VITE_AVATAR_API_URL
    .replace('http', 'ws') + `/ws/session/${sessionId}`
  
  const ws = new WebSocket(wsUrl)
  
  ws.onopen = () => {
    console.log('WebSocket 连接已建立')
  }
  
  ws.onerror = (error) => {
    console.error('WebSocket 错误:', error)
  }
  
  return ws
}
```
